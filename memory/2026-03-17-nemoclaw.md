assistant: **better-auth** sadece frontend değil, **full-stack** auth çözümü.

**Nasıl çalışır:**
- Backend'inle birlikte çalışan bir **API server** kuruyorsun
- Senin mevcut backend'ine (Laravel) **proxy** yapıyor veya **JWT üretiyor**
- Frontend'de hazır React/Vue/Svelte provider'ları var

**2 tip çalışma modu:**

1. **Session-based** (default) — Kendi session yönetimi var, cookie-based
2. **JWT plugin** — Harici servisler için JWT üretiyor

**Senin Laravel backend'ine nasıl bağlanır?**

```js
// Frontend'de
const signIn = betterAuth.signInWithOAuth({
  provider: "google", // vs
})

// Backend'de (middleware)
const session = await betterAuth.api.getSession({ headers })
// session.user içinde user bilgisi var
```

Yani **kendi backend'ini** hâlâ kullanıyorsun — ama auth kısmını better-auth hallediyor, sonra Laravel'e session bilgisini aktarıyorsun.

Senin için avantajı: Google/GitHub OAuth, magic link, passkey gibi şeyleri sıfırdan yazmak yerine hazır alıyorsun. Laravel'de sadece "bu user session'ı geçerli mi?" kontrolü yapman yeterli.
